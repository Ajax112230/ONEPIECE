# 中国传统色 · 姓名寻色 — 设计文档

## 概述

一个纯前端单页面网站，用户输入姓名后经过确定性哈希算法映射到中国传统色库中的某一个颜色，展示带有水墨风格的大色卡、色名、HEX/RGB 值和对应古诗词。

## 项目结构

```
traditional-color/
├── index.html
├── css/
│   └── style.css
├── js/
│   ├── colors.js        # 150+ 传统色数据 + 古诗词
│   ├── hash.js          # 姓名 → 色库索引算法
│   ├── particles.js     # Canvas 水墨粒子动画
│   └── main.js          # 主逻辑：输入 → 渲染色卡
```

## 技术栈

- HTML + CSS + Vanilla JS
- Canvas 2D（粒子背景动画）
- 零构建工具，直接浏览器打开

## 核心数据流

```
用户输入姓名 → hash(姓名) → index % 色库长度 → 取出颜色数据 → 渲染色卡
```

### 哈希算法（hash.js）

- 输入：任意中文字符串（姓名）
- 输出：0 到 色库长度-1 的稳定整数
- 算法：`djb2` 变种 —— `hash = 5381; for char in str: hash = (hash * 33 + charCode(char)) & 0x7fffffff`，最终 `hash % palette.length`
- 同一个名字 → 永远同一个索引
- 相近名字（差一字）→ 完全不同的颜色
- 边界处理：
  - 空输入 → 默认展示一个随机种子色（页面加载时的占位色）
  - 含英文/数字 → 同样支持，charCodeAt 对所有字符有效
  - 姓名长度无上限，但推荐 2-4 字

### 色库数据（colors.js）

约 150-180 个精选中国传统色，每个包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| name | 中文色名 | 胭脂 |
| hex | HEX 色值 | #9B2D30 |
| r, g, b | RGB 分量 | 155, 45, 48 |
| poem | 配诗一句 | 胭脂泪，相留醉，几时重 |
| poet | 作者/出处 | 李煜《相见欢》 |
| dynasty | 朝代 | 五代 |

色库按色调分组（赤、橙、黄、绿、青、蓝、紫、褐、黑白灰），每组 15-25 色。

### 水墨粒子背景（particles.js）

- 复用 `daomu-map` 的 Canvas 粒子系统设计
- 暗色画布底（#1a1a1a 或深墨色）
- 粒子低密度漂浮，约 60-80 个
- 粒子颜色根据当前展示的颜色色相微调
- 入场时粒子从四周汇聚，切换颜色时粒子有短暂涟漪效果

### 主交互流程（main.js）

1. **初始状态**：中央大字 "请君留名"，下方输入框，背景粒子浮动
2. **输入阶段**：用户输入姓名，支持 2-4 个中文字符
3. **触发查询**：按 Enter 或点击"探色"按钮
4. **过渡动画**：粒子加速汇聚，画面淡出
5. **色卡展示**：
   - 大色块从中央扩散占满视口 60-70%
   - 色名大字（毛笔质感）渐显于色块之上
   - HEX/RGB 小字显示在右下
   - 古诗词淡入出现在底部
   - 粒子背景以当前色相重新分布
6. **返回/重试**：色卡底部有"再探一色"按钮，返回输入界面

## 视觉风格

- **色调**：深墨色底 + 当前传统色作为主视觉
- **字体**：色名大标题使用 Google Fonts 的 ZCOOL QingKe HuangYou（与 daomu-map 相同，已有使用经验）或 Noto Serif SC；正文用系统无衬线
- **动画**：水墨晕染扩散效果（CSS radial-gradient transition）
- **笔触感**：输入框有毛笔勾边效果，按钮带印章感
- **粒子**：60-80 个低密度 float 粒子，类似墨滴漂浮

## 非功能需求

- 纯静态，双击 index.html 即可运行
- 所有资源本地化（不依赖 CDN，除字体外）
- 响应式设计，移动端适配
- 性能：粒子动画控制在 30fps 以保证移动端流畅

## 不做的（YAGNI）

- 色库浏览页
- 分享/保存图片
- 排行榜
- 服务端
- 用户账户
- 配色推荐
